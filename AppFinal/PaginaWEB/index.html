¿Qué hace este proyecto? ML aplicado

El objetivo es construir un estimador de precios de vehículos usados integrando:

  • Un dataset real de coches (limpieza y preparación en Python).
  • Entrenamiento de un modelo Random Forest para predecir el precio.
  • Guardado del modelo en un archivo .joblib.
  • Despliegue de una API con FastAPI que expone el endpoint /predict.
  • Esta página web en JavaScript que consume la API y muestra el resultado.
Python FastAPI scikit-learn Joblib REST API
¿Cómo se manejan los datos faltantes? Median Imputation

Para no obligar al usuario a capturar todas las características técnicas del vehículo, varios campos numéricos se consideran opcionales.

Cuando se envía la petición a la API, si uno de esos valores llega como null, el backend rellena ese campo usando la mediana de esa columna en el dataset original (ddf4_limpio.csv).

De esta forma, el modelo sigue funcionando incluso con información incompleta, manteniendo la entrada coherente con los datos de entrenamiento.
